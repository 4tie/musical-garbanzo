# Robustness Validation

## Purpose

Robustness validation checks whether a strategy's evidence remains usable when compared across baseline, out-of-sample, walk-forward, and sensitivity evidence.

A strategy is not considered validated because one backtest looked good. Robustness checks look for fragile behavior such as trade count collapse, profit factor collapse, expectancy turning negative, drawdown expansion, unstable WFO windows, too few trades, and missing critical metrics.

Robustness evaluation uses already-collected metrics. It does not run Freqtrade, edit strategies, approve strategies, export strategies, or generate trading recommendations.

## Checks

The initial evaluator includes these checks:

- Trade count stability: fails when OOS trades collapse materially from baseline, fails when OOS trades are below the minimum evidence floor, and critically fails when OOS trades are zero.
- Profit factor stability: fails when OOS profit factor collapses materially from baseline and critically fails when OOS profit factor is missing or not above 1.
- Expectancy stability: fails when OOS expectancy collapses materially from baseline and critically fails when OOS expectancy is missing or not positive.
- Drawdown stability: fails when OOS drawdown expands too much from baseline.
- WFO stability: fails when supplied WFO pass rate is below policy, WFO windows have zero trades, WFO expectancy is not positive, WFO profit factor is not above 1, or WFO drawdown is unstable.
- Sensitivity variants: fails when variant pass rate is below policy or any variant has missing critical metrics, zero trades, non-positive expectancy, profit factor not above 1, or drawdown above policy.

## Severity Rules

Severity levels:

- `info`: no blocking finding.
- `warning`: material degradation that should be reviewed before continuing.
- `error`: robustness failure that should reject or require more evidence.
- `critical`: unusable evidence, such as missing critical metrics, zero OOS trades, non-positive OOS expectancy, profit factor not above 1, or WFO pass rate below policy.

Check statuses:

- `passed`: no issues were found.
- `warning`: at least one warning was found and no error or critical issue was found.
- `failed`: at least one error or critical issue was found.

## Limitations

Robustness validation is only as strong as the evidence supplied to it. It does not create evidence, download data, run Freqtrade, repair strategies, or infer missing metrics.

Sensitivity checks only evaluate provided variant results. If variants were not safely generated by earlier validation steps, the evaluator cannot treat sensitivity as proven.

## No Guarantee

Robustness validation is an evidence quality gate. Passing robustness checks does not guarantee future profit, live trading safety, or suitability for any account.
