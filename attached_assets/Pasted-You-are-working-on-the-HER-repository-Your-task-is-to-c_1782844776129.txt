# HER Core Identity

HER is a profitable-strategy discovery and validation system.

It tries to find profitable trading strategies, but it never claims profitability without evidence.

Core rule:

AI suggests.
Backend validates.
Freqtrade tests.
HER decides.

HER is not a casino-style profit promise app.

HER is an evidence-driven local trading strategy lab.

# Important Indexing Requirements

The documentation must act like an index for future AI agents.

Do not write vague docs.

Use tables where helpful.

For API index, include exact file paths.

For workflow index, include exact frontend and backend files.

For code map, include exact responsibilities.

Add “do not break” notes where needed.

Examples:

* `ValidationExecutionService` must not approve/export/live trade.
* `OptimizationRepository` must be used for `optimization_run` source resolution.
* `optimized_run_id` is the candidate for optimization validation, not `baseline_run_id`.
* Any run action must require user confirmation.
* Frontend must not show fake charts.
