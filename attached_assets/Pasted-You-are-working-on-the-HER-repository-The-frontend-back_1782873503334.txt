# Preferred Visual Style

Use a premium dark theme.

Suggested direction:

* background: deep charcoal / dark navy
* surfaces: layered dark cards
* borders: subtle low-contrast borders
* accent: cyan / electric blue / violet-blue
* success: controlled green
* warning: amber
* danger: clean red
* neutral: slate gray
* typography: clean, modern, readable
* cards: soft radius, clear spacing, not too glassy
* layout: spacious but information-dense
* charts: calm and readable
* buttons: clear hierarchy, not oversized
* badges: compact and consistent

Avoid loud gradients everywhere.

Avoid overusing glow effects.

Use visual polish, but keep the product serious.

# Required Main UX Improvement: Strategy Journey Page

Create or significantly improve:

This should be the main cockpit page.

It should not be a fake dashboard.

It must use real backend data only.

The page should include:

## 1. Hero Strategy Panel

Show:

* selected strategy if available
* readiness status
* current lifecycle state
* latest baseline status
* latest optimization status
* latest validation status
* next safe action

If no strategy is selected, show a clean empty state:

“Select a strategy from Strategy Workspace to begin.”

Do not fake a selected strategy.

## 2. Journey Timeline

A beautiful stepper/timeline:

* Strategy selected
* Readiness checked
* Baseline
* Optimization
* Validation
* Final decision

Each step should show:

* status
* short explanation
* link to detail page if available
* next action if allowed

## 3. Live Workflow Panel

If an active run exists, show:

* current stage
* running/completed/failed steps
* last updated
* controlled failure if any
* stage progress only if derived from real stages

If no active run exists, show:

“No active run.”

## 4. Evidence Summary

Show compact cards for:

* profit factor
* expectancy
* drawdown
* trade count
* validation status
* confidence score if available

Only show real values.

If missing, show “Not available yet.”

## 5. Next Action Panel

A clear panel that tells the user what to do next:

Examples:

* Select a strategy
* Fix readiness issues
* Run baseline
* Review baseline result
* Run optimization
* Review best trial
* Run validation
* Review validation evidence

No auto-execution.

Every run action requires confirmation.

# Safety Language

Keep HER honest.

Allowed language:

* Candidate
* Promising
* Rejected
* Needs validation
* Validated by evidence
* Insufficient evidence
* Improving
* Weak evidence
* Controlled failure

Forbidden language:

* Guaranteed profitable
* Approved for live trading
* Will make money
* Safe to trade live
* Profit guaranteed
* One-click trading

Always keep this disclaimer visible on validation-related pages:

“Validation is evidence only. It is not strategy approval, export, live-trading authorization, or a guarantee of future performance.”
